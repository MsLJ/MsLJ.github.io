잘 돌아가던 서버가 갑자기 멈추거나 디스크 공간이 100% 가득 차서 당황한 적이 있으신가요? 

서버에서 `df -h` 명령어로 디스크 사용량을 점검해 보니, 다른 곳도 아닌 `/var/lib/docker/...` 하위 경로가 용량을 전부 잡아먹고 있었습니다. 원인을 추적해 보니 Docker 컨테이너가 뱉어내는 **로그 파일(`json.log`)**이 범인이었습니다.

이번 글에서는 제가 겪은 Docker 디스크 풀(Disk Full) 문제의 원인과, 이를 해결하기 위해 취한 조치 내용을 정리해 보았습니다.

---

## 1. 문제 원인: 가득 찬 `*-json.log`

Docker는 컨테이너 내부 애플리케이션의 표준 출력(stdout/stderr)을 호스트 서버의 특정 경로에 JSON 형태로 차곡차곡 저장합니다. 

보통 아래와 같은 경로에 로그가 쌓이게 됩니다.
* `/var/lib/docker/containers/컨테이너ID/컨테이너ID-json.log`
* (혹은 `overlay2` 스토리지 레이어 내부)

애플리케이션에서 무한 루프가 돌거나, 에러 로그가 반복해서 찍히는 경우, 또는 사용하지 않는 컨테이너가 백그라운드에서 계속 실행 중인 경우 이 로그 파일이 수십 GB까지 커지면서 서버 디스크를 마비시킵니다.

---

## 2. 해결 방법: 안 쓰는 컨테이너 정리 (`docker rm`)

주기적으로 로그를 지우는 것도 방법이겠지만, 제 경우에는 **사용하지 않는 컨테이너가 종료되지 않고 계속 살아있어서 로그를 끊임없이 구워내고 있는 것**이 근본적인 문제였습니다.

심지어 컨테이너가 `Exited` 상태로 멈춰 있더라도, 완전히 삭제하지 않으면 기존에 쌓인 로그 파일은 디스크에 그대로 남아 용량을 차지합니다. 

따라서 아래와 같은 순서로 조치하여 용량을 확보했습니다.

### ① 문제의 컨테이너 중지
먼저 무의미하게 구동 중이던 컨테이너를 중지시켰습니다.
```bash
docker stop <컨테이너_ID>
② 컨테이너 완전 삭제를 통한 용량 확보
컨테이너를 삭제하면 해당 컨테이너와 연결되어 있던 고용량의 json.log 파일과 임시 레이어들이 한 번에 깔끔하게 날아갑니다.

Bash
docker rm <컨테이너_ID>
💡 Tip. 종료된 모든 컨테이너 한 번에 지우기
만약 멈춰 있는 안 쓰는 컨테이너가 너무 많다면 아래 명령어로 한 번에 정리할 수 있습니다.

Bash
docker container prune

## 3. (참고) 실행 중인 컨테이너의 로그만 급하게 비워야 할 때

만약 서비스가 가동 중인 컨테이너라 `docker rm`으로 삭제할 수 없는 상황이라면, 로그 파일 자체를 `rm` 명령어로 강제 삭제하면 안 됩니다. Docker 프로세스가 파일 핸들을 놓치지 않아 용량이 안 줄어들거나 에러가 날 수 있기 때문입니다.

이때는 파일의 형태는 유지하되 **내용물만 0바이트로 비워주는 명령어**를 사용해야 합니다.

```bash
# 특정 컨테이너의 로그만 비우기
truncate -s 0 /var/lib/docker/containers/컨테이너ID/*-json.log

# 모든 컨테이너의 로그를 한 번에 비우기 (root 권한 필요)
sudo sh -c "truncate -s 0 /var/lib/docker/containers/*/*-json.log"
4. 재발 방지를 위한 근본적인 해결책: Log Rotation 설정
매번 디스크가 터질 때마다 수동으로 컨테이너를 지우거나 로그를 비울 수는 없습니다. Docker 자체 설정을 통해 로그 파일의 최대 크기를 제한하는 것이 가장 좋습니다.

/etc/docker/daemon.json 파일을 열어(없으면 생성) 로그 로테이션 설정을 추가해 줍니다.

JSON
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
max-size: 로그 파일 한 개당 최대 크기 (예: 10MB)

max-file: 보관할 로그 파일의 최대 개수 (예: 3개)

이렇게 설정하면 로그가 10MB를 넘었을 때 오래된 로그를 지우며 순환하기 때문에, 하나의 컨테이너가 디스크를 전부 가득 채우는 불상사를 원천 차단할 수 있습니다. (설정 후 sudo systemctl restart docker로 도커 재시작 필요)

맺음말
Docker 컨테이너는 편리하지만, 제대로 관리하지 않고 방치하면 json.log나 사용하지 않는 레이어들 때문에 서버가 쉽게 터질 수 있습니다. 주기적으로 docker ps -a로 컨테이너 상태를 점검하고, Log Rotation을 필수로 적용해 두는 습관이 중요한 것 같습니다.
