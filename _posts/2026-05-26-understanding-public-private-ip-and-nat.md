우리가 매일 사용하는 인터넷, 그리고 개발할 때 자주 접하는 가상머신(VM)과 도커(Docker). 이들의 네트워크를 이해하기 위한 가장 기본이 되는 개념이 바로 **공인 IP(Public IP)**와 **사설 IP(Private IP)**입니다. 

오늘은 이 둘의 차이점과 라우팅의 원리, 그리고 이들이 서로 통신할 수 있게 해주는 NAT 기술까지 쉽고 명확하게 정리해 보겠습니다.

---

## 1. 사설 IP (Private IP) 란?

사설 IP는 쉽게 말해 **"우리들만의 닫힌 공간(사설망)에서 쓰는 가명"**입니다. 인터넷과 직접 연결되지 않은 독립된 내부 네트워크에서 사용되며, 전 세계 모든 기기에 고유한 주소를 부여할 수 없기 때문에 특정 대역을 지정해 내부적으로만 재사용하도록 약속했습니다.

RFC 1918 규정에 따라 사설 IP로 사용할 수 있는 종류와 범위는 딱 **3가지 대역**으로 정해져 있습니다.

| 클래스 | 사설 IP 주소 범위 | 주요 용도 |
| :--- | :--- | :--- |
| **Class A** | `10.0.0.0` ~ `10.255.255.255` | 대규모 기업망, 대형 클라우드 환경 |
| **Class B** | `172.16.0.0` ~ `172.31.255.255` | 중규모 네트워크, 가상화 환경 |
| **Class C** | `192.168.0.0` ~ `192.168.255.255` | 가정용 공유기, 소규모 사무실 |

이 사설 IP들은 독립된 공간에서 쓰이기 때문에 **중복이 가능**합니다. 예를 들어 A라는 회사의 내부 PC 주소가 `172.16.11.6`이어도, B라는 다른 회사의 내부 PC 주소가 똑같이 `172.16.11.6`일 수 있습니다. 서로 네트워크가 분리되어 있다면 충돌이 일어나지 않고 각 서버마다 자유롭게 할당할 수 있습니다.

---

## 2. 공인 IP (Public IP) 란?

공인 IP는 **"인터넷 세상에서 유일무이한 진짜 주소(주민등록번호)"**입니다. 우리가 흔히 아는 포털 사이트나 구글 서버 등, 외부 인터넷을 통해 접속하는 모든 목적지는 이 공인 IP를 가집니다.

### ❓ 왜 공인 IP는 중복되면 안 될까요?
정답은 **라우팅(Routing)의 문제** 때문입니다. 

인터넷 세상에는 데이터의 목적지를 안내해 주는 **라우터(Router)**라는 장비들이 있고, 이 라우터들은 일종의 내비게이션 지도 책인 **'라우팅 테이블(Routing Table)'**을 가지고 있습니다. 

만약 전 세계에 동일한 공인 IP 주소가 존재한다면, 라우터들은 데이터를 정확히 어느 국가의 어느 장비로 보내야 할지 헷갈리게 되고 경로가 심하게 꼬여버립니다. 결과적으로 IP 충돌이 일어나며 정상적인 인터넷 연결이 불가능해집니다. 따라서 공인 IP는 국제기구(IANA 등)에 의해 철저하게 전 세계에서 단 하나만 존재하도록 관리됩니다.

## 3. 사설 IP가 인터넷을 하는 방법: NAT와 게이트웨이

그렇다면 회사나 집에서 사설 IP(`172.16.11.6`)를 가진 내 컴퓨터는 어떻게 외부 외부 인터넷과 통신할 수 있는 걸까요? 

여기서 **게이트웨이(Gateway)**와 **NAT(Network Address Translation)** 기술이 등장합니다.

1. **출입구 찾기 (Gateway):** 사설망 내부의 컴퓨터가 외부 인터넷으로 나갈 때, 먼저 외부망으로 통하는 문인 **게이트웨이**를 거치게 됩니다. (예: 게이트웨이의 내부 주소가 `172.16.11.1`인 경우)
2. **주소 변환 기술 (NAT):** 게이트웨이 장비 내부에서는 사설 IP를 공인 IP로 변환해 주는 **NAT** 기술이 동작합니다.
3. **가면 쓰기:** 게이트웨이는 자신이 외부망과 연결해 둔 공인 IP(예: `58.225.225.30`)로 사설 IP 주소를 싹 바꿔서 인터넷 세상으로 데이터를 내보냅니다.

결과적으로 외부 인터넷 서버가 바라볼 때는 나의 사설 IP(`172.16.11.6`)는 전혀 보이지 않고, 게이트웨이의 공인 IP(`58.225.225.30`)만 보이게 됩니다. 이 덕분에 사설 IP로도 안전하고 자유롭게 인터넷 통신이 가능해집니다.

## 4. 이미 우리는 흔하게 쓰고 있다: VM과 Docker

이 구조가 멀게 느껴진다면, 개발할 때 자주 쓰는 **VM(가상머신)**이나 **Docker(도커)**를 떠올려 보세요. 아주 좋은 예시입니다.

VM을 생성하거나 도커 컨테이너를 띄워서 내부 IP(`ifconfig` 또는 `ip addr`)를 확인해 보면, 십중팔구 `172.17.x.x` 나 `192.168.x.x` 같은 사설 IP를 할당받은 것을 볼 수 있습니다. 

그럼에도 불구하고 이 컨테이너 안에서 인터넷 패키지를 정상적으로 다운로드할 수 있는 이유는 무엇일까요? 이미 우리가 사용하는 컴퓨터(호스트 PC)나 서버가 **게이트웨이 역할**을 하며 내부적으로 **NAT 기술을 적용**해 주고 있기 때문입니다. 

컨테이너의 사설 IP는 호스트의 IP로 변환되어 외부와 통신하며, 외부에서 볼 때는 우리의 사설 IP가 아닌 변환된 공인 IP로 인식됩니다.

## 5. 💡 실무에서 바로 써먹는 네트워크 꿀팁!

### ① 내 서버의 '진짜 공인 IP' 확인하는 방법
리눅스 서버 환경에서 내부 명령어를 치면 사설 IP만 보여서 외부 방화벽을 뚫거나 API 연동을 할 때 당황할 때가 있습니다. 내 서버가 NAT를 거쳐 외부로 나갈 때 어떤 **공인 IP**로 변환되는지 확인하고 싶다면 아래 명령어를 사용해 보세요.

```bash
curl ifconfig.me

원리: 외부의 ifconfig.me 서버에 요청을 보낸 뒤, 그 서버가 "너 나한테 이 IP로 접속했어"라고 찍어주는 방식입니다. NAT를 거쳐 최종적으로 변환된 나의 공인 IP를 가장 확실하게 확인할 수 있는 방법입니다.

② SSH 원격 접속이 안 된다면? 대역과 라우팅 확인하기
새로 만든 VM이나 서버에 SSH(port 22)로 접속하려고 하는데 연결이 안 된다면, 네트워크 대역(Subnet) 문제일 확률이 높습니다.

동일한 사설망 대역에 있을 때: 내 컴퓨터가 192.168.0.10이고 서버가 192.168.0.20처럼 동일한 사설 IP 대역(동일한 공유기나 스위치 하단)에 있다면, 별도의 복잡한 설정 없이 바로 SSH 접근이 가능합니다. 외부망을 거치지 않고 내부에서 직접 통신하기 때문입니다.

다른 사설망 대역에 있을 때: 만약 회사의 내 자리(192.168.x.x)에서 IDC 센터나 클라우드에 있는 서버의 사설 IP(10.x.x.x)로 바로 SSH 접근을 하려고 하면 통신이 불가능합니다. 사설 IP는 인터넷 망을 타고 넘어갈 수 없기 때문입니다.

해결책: 이럴 때는 VPN(가상 사설망)을 연결하여 같은 사설 대역처럼 묶어주거나, 외부로 노출된 공인 IP(또는 포트 포워딩)를 주소로 지정하여 접근해야 합니다.

📌 한 줄 요약
공인 IP: 인터넷 지도(라우팅 테이블)가 찾아갈 수 있는 전 세계 유일한 주소

사설 IP: 내부망에서 중복해서 쓸 수 있는 가명 주소 (Docker, VM 등에서 활용)

NAT: 사설 IP가 외부로 나갈 수 있도록 게이트웨이에서 공인 IP로 옷을 갈아입혀 주는 기술
